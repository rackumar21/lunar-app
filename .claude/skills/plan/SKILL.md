# Planning Phase

We are in the PLANNING phase. Do not write or edit any code yet.

Exploration is done. Now we write the plan before touching anything.

## Steps

1. Re-read CLAUDE.md and docs/PLAN.md to confirm current position
2. Based on the exploration findings, write a clear step-by-step plan
3. For each step, explain:
   - What we're doing
   - Why we're doing it (not just what)
   - What could go wrong
   - How we'll know it worked
4. Call out any decisions Rachita needs to make before we start
5. Get explicit confirmation before moving to building

## Format for the plan

Present it as:

**Goal:** One sentence — what will be true when this is done?

**Steps:**
1. [Step] — [Why this step matters]
2. ...

**Risks:** What could break or go wrong
**How we'll test:** How we verify it worked (usually: run locally at localhost:5173)
**Decisions needed:** Anything Rachita needs to decide before we start

## Remember
- No code until Rachita says "yes, let's build"
- Plain English throughout — explain technical terms
- Enterprise way AND solo builder shortcut for any significant decision
- Keep plans small and focused — one thing at a time
