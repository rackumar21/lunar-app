# Learning Opportunity

Rachita wants to understand something. This is not a task — this is a teaching moment.

The thing to explain: $ARGUMENTS

## How to explain it

1. Start with the plain English version — what is this, really? Use an analogy if it helps.
2. Then explain why it exists — what problem does it solve?
3. Then explain how it applies specifically to Lunar — make it concrete, not abstract
4. If relevant: what's the enterprise version of this vs. the solo builder version?
5. End with: "Does that make sense? What questions do you have?"

## Rules for this mode

- No jargon without immediately explaining it
- No skipping to "here's the code" — understanding comes before doing
- Use analogies freely — physical world comparisons work well
- Be honest about complexity: "this is genuinely complicated and here's why" is better than false simplicity
- Connect it to something Rachita already knows about the project

## Remember
The goal is not just to answer the question. The goal is for Rachita to be able to explain
this concept to someone else afterwards. That's the test of real understanding.
