# Exploration Phase

We are in the EXPLORATION phase. Do not write or edit any code yet.

Your job right now is to fully understand the current state before anything changes.

## Steps

1. Read CLAUDE.md to orient yourself on the project and where we are
2. Read docs/PLAN.md to understand the current chapter and what we're working toward
3. Read the files most relevant to the task at hand
4. Identify: what exists, what's missing, what could break, what decisions need to be made
5. Ask Rachita any clarifying questions before proceeding — never assume

## How to report back

End your exploration with a clear summary:
- **What I found:** what the current state actually is
- **What's relevant:** which files and lines matter for this task
- **Questions before we plan:** anything that needs a decision before writing a plan

## Remember
- Rachita is a non-technical PM. Use plain English throughout.
- Never start building during exploration. That comes after /plan.
- Be specific about file paths and line numbers when referencing code.
