# End-of-session wrap-up

Do all of the following before Rachita logs off.

## 1. Update MEMORY.md
Update `/Users/rachita/.claude/projects/-Users-rachita-Work-lunar-app/memory/MEMORY.md` to reflect current app state. Replace anything outdated. Keep it under 200 lines.

## 2. Log new decisions
Append any decisions made today to `docs/DECISIONS.md`. Each entry needs: what we chose, what we rejected, and why. Number sequentially from the last entry.

## 3. Update PLAN.md
Mark any newly completed items. Add notes on what changed from the original plan.

## 4. Update README if needed
If we added a user-facing feature today, add it to the Features section. Keep it factual and brief.

## 5. Summarise the session
Write a plain-English summary covering:

**What we built today** (one line per feature/fix)

**What broke and why** — honest account of every mistake and what caused it

**What took too long and why** — the debugging rabbit holes, what we should do faster next time

**What you learned today** — concepts, not tasks. What does Rachita now understand that she didn't this morning?

**What's next** — top 2-3 things for the next session

## Rules
- Be honest about mistakes. Don't soften them.
- Connect every mistake to a root cause and a lesson.
- "What took too long" should read like a retrospective, not an excuse.
- End with: "See you tomorrow."
