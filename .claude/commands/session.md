# Session Status Check

Use this at the start of every session to orient before doing anything.

## Steps

1. Read CLAUDE.md — who is Rachita, how do we work, what are the conventions
2. Read docs/PLAN.md — what chapter are we on, what's done, what's next
3. Read docs/ARCHITECTURE.md — what does the current structure look like
4. Briefly scan the relevant source files to confirm they match the documented state
5. Report back clearly

## How to report back

**Where we are:** Which chapter, which step within the chapter
**What we did last time:** Brief summary of the last completed work
**What's next:** The immediate next step, in plain English
**Any blockers:** Anything that needs a decision or discussion before we can proceed
**Suggested plan for today:** What I'd recommend we tackle in this session and why

## Then ask

"Does this match what you were thinking? Or did you want to work on something different today?"

## Remember
- This is orientation, not execution. Don't start building yet.
- Always read the docs first — don't rely on memory of previous sessions.
- Rachita may have done things outside our sessions (deployed, changed something) — check git status too.
